# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Personal website at **adam-young.co.uk** — built with [Publish](https://github.com/JohnSundell/Publish) (John Sundell's Swift static site generator), hosted on GitHub Pages via a GitHub Actions deployment workflow.

## Commands

```bash
swift build                        # type-check / build the executable
swift run AdamYoungSite            # build the site → Output/
swift package update               # bump dependencies
python3 -m http.server -d Output 8080   # preview locally at http://localhost:8080/
```

There are no tests, linters, or formatters configured.

## Deployment

Pushing to `main` triggers `.github/workflows/pages.yml`, which runs on `macos-latest`, builds via `swift run -c release AdamYoungSite`, and deploys `Output/` via `actions/deploy-pages@v5`. The repo's GitHub Pages source must be set to **"GitHub Actions"** (not "Deploy from a branch") for the workflow to deploy.

`Package.resolved` is checked in to pin the dependency tree for CI reproducibility.

## Architecture

### Layout

- `Package.swift` — Swift package manifest. Depends on `Publish` and `SplashPublishPlugin`.
- `Sources/AdamYoungSite/`
  - `main.swift` — publishing pipeline (Splash plugin + copy resources + add markdown + generate HTML/RSS/sitemap).
  - `AdamYoungSite.swift` — `Website` conformance: defines `SectionID` (just `.blog`), site URL/name/description, and site-level constants used by the theme (Twitter handle, GA ID, etc.).
  - `Theme/Theme+AdamYoung.swift` — custom `Theme` and `HTMLFactory`. Implements `makeIndexHTML`, `makeSectionHTML`, `makeItemHTML`, `makePageHTML`. Tag pages return `nil` (not generated).
  - `Theme/Theme+AdamYoung+Components.swift` — shared Plot `Node` builders for the `<head>` (incl. SEO/OG/JSON-LD), ambient blob layer, top nav, site footer, post-meta paragraph, post-link card. Date formatting helpers live here too.
- `Content/`
  - `index.md` — title + description for home (body unused; the home layout is hardcoded in `makeIndexHTML`).
  - `blog/index.md` — Publish auto-treats this as the blog `Section` (title "Blog", description shown in the blog header).
  - `blog/<slug>.md` — individual posts.
- `Resources/` — copied verbatim into the output by `.copyResources()`. Contains `styles.css`, `404.html` (raw HTML, served by GH Pages on 404), `CNAME`, and `assets/`.
- `Output/` — generated; gitignored.

### Routing

- `/` — home (Index)
- `/blog/` — blog listing (Section)
- `/blog/<slug>/` — individual post (Item)
- `/feed.rss` — RSS 2.0 feed of `[.blog]`
- `/sitemap.xml`
- `/404.html` — raw resource (not generated by the theme)

### Design system

Visual language is "cool frosted glass" — drifting cyan/indigo/mint blobs behind glass cards with `backdrop-filter`, supporting both light and dark via `prefers-color-scheme`. All styling lives in **one file**, `Resources/styles.css`, structured as:

1. `:root` CSS custom properties for both colour schemes
2. Layout primitives (`.ambient`, `.glass`, `.nav`, `.hero`, `.cards`, `.card`, `.chip`)
3. Blog-specific styling (`.blog-header`, `.posts-list`, `.post-link`, `.post-body`)
4. Splash syntax-highlight palette at the bottom (`pre code` selectors targeting `.keyword`, `.string`, `.type`, `.call`, `.number`, `.comment`, `.property`, `.dotAccess`, `.preprocessing`)

The `.glass` class is the reusable mixin: it stacks `backdrop-filter` blur + a thin gradient border (via `mask-composite`) + a top inner highlight.

### SEO / metadata

There is no `jekyll-seo-tag` equivalent in Publish — meta tags are hand-rolled in `Theme+AdamYoung+Components.swift` (`adamYoungHead(for:on:)`):

- Plot's `.title("...")` automatically emits `<title>`, `og:title`, and `twitter:title`. Don't double-emit those.
- Hand-rolled tags cover: theme-color (light/dark), apple-mobile-web-app, description, og:type/og:site_name/og:description/og:url/og:image/og:image:alt, twitter:card/site/creator, JSON-LD `Person` schema.
- Per-post hero images are set via `image:` in the post's front matter; this drives `og:image` for that page (item's `imagePath` falls back to site `imagePath`).

### Code highlighting

`SplashPublishPlugin` (Swift-only highlighter by Sundell). Installed in the pipeline with `withClassPrefix: ""`, so token spans use bare class names (`.keyword`, `.string`, etc.) inside `<pre><code>`. Only Swift code blocks (` ```swift `) are highlighted; other languages render as plain text.

## Adding a blog post

Create `Content/blog/<slug>.md` with front matter:

```yaml
---
title: Post Title
date: 2026-04-26 12:00
description: Short blurb shown on the blog index and as og:description.
tags: tag1, tag2
image: /assets/images/posts/<slug>.svg
---

Body…
```

Notes:

- `tags` is a comma-separated string (Publish parses it into `[Tag]`).
- Don't quote the `title:` value with `"…"` — Publish's frontmatter parser is naive and the quotes would render literally.
- A title containing a colon is fine: only the **first** colon is treated as the key/value separator.
- The post URL is derived from the file name: `Content/blog/foo-bar.md` → `/blog/foo-bar/`.
- Hero images live under `Resources/assets/images/posts/` and are SVG (small, on-brand). Twitter doesn't render SVG OG cards, so prefer PNG/JPG for `image:` if Twitter sharing matters; the SVG is fine for the inline hero.

## Conventions worth knowing

- The hero "AY" avatar on the home page is a CSS gradient, not an image — `Resources/assets/images/me.jpg` is only the OG/share image and favicon.
- The home page's hero copy, projects cards, "Elsewhere" links, and tech-stack chips are **hardcoded in `makeIndexHTML`** (Plot DSL). They are layout, not content. To change them, edit `Theme+AdamYoung.swift` (`heroSection`, `projectsSection`, `elsewhereSection`, `stackSection`).
- The 404 page is **raw HTML** at `Resources/404.html`. It does not go through the theme. If the design changes, copy the relevant `<head>` and nav markup over manually.
- `Plot.Language.english` renders as `lang="en"`. The theme uses `.attribute(named: "lang", value: "en-GB")` directly to keep the original `en-GB` locale.
